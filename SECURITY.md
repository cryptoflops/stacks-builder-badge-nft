# Security Policy

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability within Builder Badge, please send an e-mail to the maintainers or open a confidential issue.

Please include the following information in your report:
- Type of issue (e.g., buffer overflow, SQL injection, cross-site scripting, etc.)
- Full path of source file(s) related to the manifestation of the issue
- The location of the affected source code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit the issue

We will acknowledge your report within 48 hours and provide a timeline for a fix within 5 business days.

## Non-Vulnerability Reports
For general bugs or performance issues, please open a regular GitHub Issue.
