The following is a set of guidelines for contributing to Builder Badge. These are mostly guidelines, not rules. Use your best judgment, and feel free to propose changes to this document in a pull request.

## How Can I Contribute?

### Reporting Bugs
Bugs are tracked as GitHub issues. When creating a bug report, please include as many details as possible:
- Use a clear and descriptive title.
- Describe the exact steps which reproduce the problem.
- Explain which behavior you expected to see and why.
- Provide specific examples to demonstrate the steps.

### Suggesting Enhancements
Enhancement suggestions are tracked as GitHub issues.
- Use a clear and descriptive title for the suggestion.
- Provide a step-by-step description of the suggested enhancement.
- Explain why this enhancement would be useful to most Builder Badge users.

### Pull Requests
- Fill in the required template.
- Do not include any private keys or sensitive data.
- Ensure all tests pass (`npm test` in root).
- Follow the existing code style.

## Styleguide

### Clarity
- Use descriptive names for variables and functions.
- Comment complex logic.
- Follow SIP-009 and SIP-010 standards where applicable.

### Frontend
- Use Tailwind CSS for styling.
- Follow React Hooks best practices.
- Ensure responsive design.

## Questions?
Feel free to open an issue or reach out to the project maintainers.
